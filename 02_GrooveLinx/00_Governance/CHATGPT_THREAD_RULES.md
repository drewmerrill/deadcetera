# ChatGPT Thread Rules

Purpose:
Prevent ChatGPT workflow chaos, mega-thread degradation, context drift, and AI performance problems.

Chats are temporary working sessions.

They are NOT permanent institutional memory.

---

# Core Rule

One thread = one operational purpose.

Do NOT mix:

* strategy
* debugging
* architecture
* random troubleshooting
* personal tasks
* UX review
* implementation details

inside giant mega-threads.

---

# Official Chat Types

## 1. Drew Command Center

Purpose:

* strategic HQ
* prioritization
* workflow governance
* AI coordination
* review
* sequencing

Avoid:

* implementation debugging
* random troubleshooting
* deep tactical coding

---

## 2. GrooveLinx Strategy

Purpose:

* product direction
* UX
* architecture
* workflow simplification
* roadmap decisions

Avoid:

* tactical implementation debugging

---

## 3. Tactical Build Threads

Examples:

* Spotify Reliability Sprint
* Mobile Performance
* Rehearsal Intelligence
* Scheduling Engine

Purpose:
Focused execution discussions.

Keep narrow and tactical.

---

## 4. Scratchpad

Purpose:

* temporary exploration
* random ideas
* throwaway discussions

Nothing important should permanently live here.

---

# When To Start A New Thread

Immediately create a continuation thread if:

* responses become repetitive
* chat becomes laggy
* multiple domains become mixed
* AI quality degrades
* thread becomes mentally exhausting
* implementation context becomes bloated

Do NOT try to rescue giant degraded threads forever.

---

# Thread Closeout Procedure

Before abandoning a major thread:

1. Run institutional memory extraction
2. Create handoff summary
3. Save important continuity into repo docs
4. Start clean continuation thread

---

# Important Distinction

Chats:

* temporary cognition
* active reasoning
* exploration

Repo docs:

* institutional memory
* continuity
* durable truth

GitHub Projects:

* execution state

---

# Operational Goal

Keep AI systems:

* fast
* coherent
* focused
* strategically aligned
* low-chaos
* scalable
