## Summary

Resume prompt on app load, now-playing bar when navigating away, cache invalidation on YouTube errors. Playback feels continuous.

| File | What |

## Resume UI

**Trigger:** On app load (3s delay), if `gl_player_state` exists and < 24h old

**Display:** Fixed bottom prompt:
> Resume setlist?
> ▶ Continue from Sugaree (7 of 15)
> [Resume] [✕]

**Resume action:** Loads setlist from Firebase, launches player at saved song index

**Dismiss:** Removes prompt + clears saved state

**State shape** (updated with song title + total):
```json
{ "setlistId": "...", "setlistName": "...", "songIdx": 7, "songTitle": "Sugaree", "total": 15, "ts": ... }
```

## Now-Playing Bar

**Trigger:** When user closes the player overlay (✕ button) while a queue exists

**Display:** Fixed bottom bar:
> ▶ Sugaree
> 7 of 15 · Tap to return [✕]

**Tap:** Removes bar, reopens full player overlay, continues from current song

**Close (✕):** `fullClose()` — destroys player, removes bar, clears queue entirely

**Two close modes:**
- `close()` — hides overlay, shows now-playing bar (soft close)
- `fullClose()` — destroys everything (hard close)

## Cache Invalidation

**On YouTube error** (`onError` event):
1. Removes the failed video ID from `gl_yt_id_cache` for that song
2. Clears `song.youtubeId` in the queue
3. Shows fallback UI

**Effect:** Next time the song is played, resolution runs fresh (checks versions, potentially finds a different/updated video).

## Edge Cases

- **No resume state:** Prompt never shows
- **Resume state expired (>24h):** Cleared silently, no prompt
- **Setlist deleted since last play:** "Setlist not found" toast, state cleared
- **Now-playing bar + back-to-feed bar:** Both fixed-bottom, now-playing has higher z-index (9500 vs 900)
- **Multiple close/reopen:** Now-playing bar prevents duplicates via ID check
- **fullClose from now-playing bar:** Uses `event.stopPropagation()` so bar click doesn't re-open
