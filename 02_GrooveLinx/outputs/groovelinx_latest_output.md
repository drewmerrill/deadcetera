## Resume Logic

| State Age | Behavior |
|-----------|----------|
| < 2 hours | Auto-resume immediately — no prompt, no decision |
| 2–24 hours | Show resume prompt (existing UI) |
| > 24 hours | State expired, silently cleared |

**Auto-resume flow:**
1. `DOMContentLoaded` → `requestAnimationFrame`
2. Check `gl_player_state` age
3. If < 2h: call `_resumeFromState()` directly
4. Player launches at saved song index
5. Toast: "▶ Resumed: Sugaree (7/15)"

**Manual resume (2-24h):**
- Same as before: floating prompt with "Resume" button

## Confidence Signal

On every successful resume (auto or manual):

"▶ Resumed: Sugaree (7/15)"

Shows after player launches (via `showToast`), confirming exactly where playback picked up.

## Edge Cases

- **App opened < 2h after pause:** Auto-resumes silently. User sees player open + toast.
- **App opened 3h after pause:** Prompt appears. User chooses to resume or dismiss.
- **App opened 25h later:** State expired. Nothing happens.
- **Firebase offline during auto-resume:** "Could not load setlist" toast. No crash.
- **Setlist deleted since pause:** "Setlist not found" toast, state cleared.
- **Auto-resume + page navigation:** Player overlay opens on top of whatever page is showing (z-index 10000).
