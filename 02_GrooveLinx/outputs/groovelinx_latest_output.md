# GrooveLinx — P0 Fix: Blank Screen on iPad PWA

_Generated by Claude · 2026-03-24_

## Exact Page Affected

**Any page** — the blank screen is not page-specific. It's a **boot-level failure** where no `showPage()` call ever executes, leaving all `.app-page` elements hidden.

## Root Cause

The app startup has **three independent code paths** that must each end with a `showPage()` call. Two of them had gaps where failures would silently bail out without showing any page:

### Gap 1: `initFirebaseOnly()` rejects (offline, network error)
- The `.catch()` block loaded songs from localStorage but **never called `showPage()` or `glHeroCheck()`**
- Result: hero hidden (user was signed in), all pages hidden → blank screen

### Gap 2: `handleGoogleDriveAuth(true)` — GIS script fails to load
- `loadGoogleDriveAPI()` tries to load `https://accounts.google.com/gsi/client`
- On iPad PWA with spotty connectivity, this can fail or hang
- The catch block at line 5857 logged the error and **returned without showing any page**
- If `glLastPage` was null (no page restore pending), nobody ever called `showPage()`

### Gap 3: No universal safety net
- Multiple timers (800ms page restore, 900ms hash restore) could all miss if conditions weren't met
- `_glPageRestorePending` was never cleared after use
- No fallback existed for "nothing worked, show something"

## Fixes Applied

### 1. Firebase init failure now shows a page (`app.js`)
```
.catch(err => {
    // ... existing localStorage fallback ...
    // NEW: still show a page
    glHeroCheck(!!savedEmail);
});
```

### 2. GIS failure now restores session from cache (`app.js`)
When `loadGoogleDriveAPI()` fails in silent mode:
- Restores user identity from localStorage (email, name, picture)
- Hides hero
- Calls `showPage()` with saved or default page
- Falls back to `glHeroCheck(false)` if no cached session

### 3. Boot Watchdog — last-resort blank screen prevention (`index.html`)
5-second watchdog timer after DOMContentLoaded:
- Checks if ANY `.app-page` is visible OR hero is visible
- If nothing is visible → forces `showPage('home')`
- Absolute fallback: manually un-hides `#page-home` if `showPage` isn't available
- Logs `🚨 Boot watchdog` to console for diagnostics

### 4. Missing page warning (`navigation.js`)
`showPage()` now logs a warning if the target `#page-{name}` element doesn't exist in the DOM.

### 5. `feed` added to hash router valid pages (`navigation.js`)
Was already in the VALID restore list but missing from `_HASH_VALID_PAGES`.

## PWA-Specific or Page-Specific?

**PWA-specific.** The blank screen only occurs when:
- Google Identity Services script fails to load (common on iPad PWA with intermittent connectivity)
- OR Firebase init fails (offline launch)
- AND no page restore timer fires (e.g., `glLastPage` is null or invalid)

In a browser tab, GIS almost always loads. On iPad PWA, the installed app can launch before network is fully available.
