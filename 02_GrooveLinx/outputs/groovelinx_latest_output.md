# GrooveLinx ‚Äî Band Feed v3: Actionable Workflow Surface

_Generated by Claude ¬∑ 2026-03-24_

## What Changed

Upgraded Band Feed from a **read-only aggregation view** to a **full actionable workflow surface**. Every feed item is now clickable with inline actions ‚Äî no new pages, no chat, no messaging.

## New Capabilities

### 1. Click-Through Navigation
Every feed item is clickable and routes to its source:
- **Song moments / rehearsal notes** ‚Üí Songs page with the song auto-selected
- **Ideas / polls** ‚Üí Band Room (Ideas) page
- Click target is the header and body text; action buttons are excluded from click-through

### 2. Inline Actions (per item)
Each feed item now has an action bar with:
- **Resolve / Unresolve** ‚Äî toggle resolved state (dims the card)
- **Tag Cycle** ‚Äî cycles through: FYI ‚Üí Needs Input ‚Üí Critical ‚Üí Fun
- **Add Note** ‚Äî expands an inline text input, saves a timestamped note with author attribution
- **Archive** ‚Äî hides from main feed (doesn't delete source data)

### 3. Feed Metadata Layer
New `feed_meta` path in Firebase stores overlay data per item:
- `archived` (boolean) ‚Äî hides item from active views
- `resolved` (boolean) ‚Äî overrides source data's resolved state
- `tag` (string) ‚Äî overrides source data's tag
- `notes[]` ‚Äî array of `{ text, by, ts }` objects

This means the original data (ideas, polls, moments) is **never modified** ‚Äî feed_meta is a non-destructive overlay.

### 4. Archive View
New "Archived" filter tab with count. Archived items show a "Restore" button instead of "Archive". Archived items are excluded from all other filter counts.

### 5. Notes Display
Existing notes appear inline on the feed item card, showing author, text, and relative time. New notes are added via inline input with Enter key or Save button support.

### 6. Resolved Styling
Resolved items render at reduced opacity (0.6) for visual de-emphasis while remaining accessible. Archived items render at 0.45 opacity.

## Architecture

- **No new pages** ‚Äî all actions happen inline on the feed
- **No source data mutation** ‚Äî feed_meta is a separate Firebase path that overlays display state
- **Item key format**: `{type}:{id}` (e.g., `idea:abc123`, `poll:xyz789`)
- **Firebase path**: `bands/{slug}/feed_meta/{type}:{id}/`
- **XSS protection**: `_feedEsc()` now escapes quotes for safe attribute injection
