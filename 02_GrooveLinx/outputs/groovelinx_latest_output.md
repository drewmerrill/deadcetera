## What Changed (3 files, ~80 lines)

### 1. Play Dashboard — Guided (home-dashboard.js)

Added `_renderNextActionCard` + `_renderBandScorecard` to `_renderPlayDashboard()`.

**Before**: Play dashboard had Action Owed → Listening Card → Upcoming Set → Readiness. Passive.

**After**: Next Action Card → Band Scorecard → Listening Card → Upcoming Set → Readiness → Action Owed. Guided.

Same cards already built for Sharpen/Lock In. Now all three modes have the decision engine.

### 2. Live Gig Mode + Float Audio (live-gig.js)

Added 🎧 button to Live Gig header. `lgToggleAudio()`:

- Builds song queue from the current setlist
- Launches `GLPlayerUI.showFloat()` — mini-player floats over charts
- Charts remain scrollable underneath
- Songs auto-advance in the audio player

**Sync**: Live Gig ↔ Audio Player stay in sync:
- Tap Next/Prev in Live Gig → audio player jumps to same song
- Audio auto-advances → Live Gig chart updates to match
- Exit Live Gig → audio player closes automatically

**Button state**: 🎧 button highlights indigo when audio is active.

### 3. Player Conflict Fix (gigs.js)

`gigPlaySetlist()` now uses GLPlayerEngine first, falls back to SetlistPlayer only if unified engine unavailable. Prevents two overlays at z-index 10000.
