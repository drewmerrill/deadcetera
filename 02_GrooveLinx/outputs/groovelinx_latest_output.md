## Toast Logic — No Stacking

**Before:** Generic toast + reinforcement toast 2.5s later (two toasts)
**After:** Reinforcement toast REPLACES generic toast on first actions

| Action | First time | Subsequent |
|--------|-----------|------------|
| Quick post | "🎉 First post — you're using Band Feed" | "Shared with the band" |
| Vote | "🎉 First vote — the band sees your input" | "✅ Vote recorded" |
| Vote on targeted item | (same first-time) | "🎯 Nice — you handled something assigned to you" |
| All items cleared | "💪 All clear — you're locked in" | (momentum nudge) |
| Return to feed | "← Back to Feed — you're in the right place" | (none, once per session) |

## New Reinforcement Triggers

| Trigger | Toast | Frequency |
|---------|-------|-----------|
| Complete targeted item | "🎯 Nice — you handled something assigned to you" | Once per session |
| Return from deep page | "← Back to Feed — you're in the right place" | Once per session |

**Frequency logic:**
- `first_post`, `first_vote`, `all_clear` — permanent milestones, fire once ever (localStorage)
- `targeted`, `return` — session milestones, fire once per session (sessionStorage)

## Edge Cases

- **First vote on targeted item:** First-vote reinforcement wins (permanent milestone > session milestone)
- **Return without prior deep-link:** `_feedBackToFeed` only fires from the back bar, which only appears after feed navigation
- **Multiple returns same session:** Only first shows toast (sessionStorage guard)
- **First-vote message updated:** "the band sees your input" (action-oriented, not "you're using Band Feed")
