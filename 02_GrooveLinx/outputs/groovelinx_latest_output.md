## Architectural Changes

### 1. FeedActionState (`js/core/feed-action-state.js`)

**Identity resolution** — single source, 4 formats unified:
- `getMyMemberKey()` — member key (`drew`, `chris`)
- `getMyDisplayName()` — display name (`Drew`, `Chris`)
- `getMyVoteKey()` — vote key (= display name, matches _bcGetName() chain)
- `getMyEmail()` — email
- `isMe(authorStr)` — fuzzy match against all formats

**Action state** — normalized per-item:
```
getActionState(item, meta) → {
  needsMyInput, needsBandInput, completeForMe, completeForBand,
  waitingOnOthers, isResolved, isArchived, iVoted,
  priorityBucket, badge: {text, tone}, cta: {label, action}
}
```

**Aggregate queries:**
- `computeSummary(items, metaMap)` → `{total, critical, needsMyInput, waitingOnBand, resolved, allClear}`
- `sortByPriority(items, metaMap)` — sorts by action urgency bucket, then chronology

**Badge rendering:**
- `renderBadgeHTML(badge)` — tone-mapped HTML with consistent styling across the feed

**Priority buckets** (lower = more urgent):
1. CRITICAL — mission_critical tag
2. NEEDS_MY_INPUT — I owe action
3. WAITING_ON_BAND — I acted, others haven't
4. RECENT_FYI — no action needed
5. RESOLVED — complete
6. ARCHIVED — hidden

**Return context** — rich object in sessionStorage:
- `{filter, focusItem, ts}` — preserves which tab and which item to scroll to
- `setReturnContext()`, `getReturnContext()`, `clearReturnContext()`, `hasReturnContext()`

### 2. Band Feed v5 Changes

**Removed from band-feed.js** (now in FeedActionState):
- `_feedGetMyVoteKey()` — replaced by `FAS.getMyVoteKey()`
- `_feedNeedsMyInput()` / `_feedNeedsBandInput()` — replaced by `FAS.getActionState().needsMyInput`
- Tag badge rendering logic — replaced by `FAS.renderBadgeHTML()`
- Note author resolution — uses `FAS.getMyDisplayName()`

**New behavior:**
- Feed loads in priority order (action urgency), not pure chronology
- Return to feed restores previous filter tab
- Return to feed scrolls to the item you clicked away from (via `focusItem`)
- "All" view groups: Critical → You Owe Input → Waiting on Band → Recent
- Filter tabs: "I Owe" / "Waiting" (shorter, action-oriented)
- Subtitle: "What do you owe? What's waiting on others? What changed?"

## UI/Behavior Changes

- **Attention bar**: "You owe input" instead of "Need Your Input" — more direct
- **Section headers**: "YOU OWE INPUT" / "WAITING ON BAND" — personal ownership language
- **Filter tabs**: Simplified to "I Owe (N)" and "Waiting (N)" — fewer words, clearer intent
- **Removed "My Items" and "Needs Band Input" redundant filters** — consolidated into "Waiting"
- **Badge on polls**: "You voted · waiting on band" (indigo) vs "Your input needed" (yellow)
- **All clear state**: "nothing needs your attention" — personal framing

## Risks / Follow-up

1. **Vote key format**: Votes stored under display names ("Drew"), not member keys ("drew"). If `_bcGetName()` ever changes its resolution chain, vote matching will break. Document this as a migration candidate.
2. **Idea completion**: Ideas are currently always `needsMyInput=true` until resolved. No per-user response tracking exists for ideas. A future enhancement could check if the current user has posted a note.
3. **FeedActionState is consumed only by band-feed.js today**. Other surfaces (home dashboard, left rail badge) could use `computeSummary()` for consistent counts.
