## Spotify State Model (4 states)

`PlaybackSession.getSpotifyState(bundleType)` returns:

| State | Label | Style |
|-------|-------|-------|
| `disconnected` | "🎵 Connect Spotify" | Green prominent |
| `connected_no_playlist` | "🎵 Create Playlist" | Green prominent |
| `synced` | "✅ Synced to Spotify" | Green dim (success) |
| `unavailable` | "🎵 Spotify" | Neutral |

Single resolver — all UI calls `getSpotifyLabel(bundleType)` + `getSpotifyStyle(bundleType)`.
GLStore bridge: `GLStore.getSpotifyState()`, `GLStore.getSpotifyLabel()`.

## Playback State Machine

```
idle → preparing → ready → starting → playing
                                    ↘ fallback
                         ↘ error
```

Every playback attempt resolves to one of: `playing`, `fallback`, `error`.
No indefinite loading states. `getState()` returns current state at any time.

## Help System

`PlaybackSession.shouldShowHelp(surface)` — returns true if not seen
`PlaybackSession.dismissHelp(surface)` — marks seen permanently
`PlaybackSession.resetHelp(surface)` — allows re-trigger
`PlaybackSession.showSurfaceHelp(surface, text, targetId)` — shows via spotlight or toast

Stored in `localStorage('gl_help_flags')`: `{ "listen_learn": true, "feed": true }`

## Error Messages

| Scenario | Message |
|----------|---------|
| System not loaded | "⚠️ Playback system not available" |
| Empty bundle | "⚠️ No songs found for rehearsal" |
| No URL | "⚠️ No playable URL for this track" |
| Open failed | "⚠️ Couldn't start playback — try opening manually" |
| No matches | Fallback overlay with YouTube/Spotify/Archive |
