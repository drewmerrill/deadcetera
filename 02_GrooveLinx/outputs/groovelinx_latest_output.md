## Version Caching

**`gl_yt_id_cache`** in localStorage: `{ "Althea": "abc123", "Sugaree": "xyz789" }`

Resolution priority:
1. **Local cache** (instant, no network)
2. **Saved versions** (Firebase, checks platform='youtube')
3. **Default North Star** (Firebase, extracts YouTube ID from URL)

Once found, ID is cached permanently. Never re-searched for the same song.

## Play From Any Song

`launch(setlistObj, name, startIdx)` now accepts optional `startIdx`. `playFromIndex(idx)` available after launch for mid-setlist jumping.

## Persistence

Stored in `localStorage('gl_player_state')`:
```json
{ "setlistId": "Summer Tour 2026", "setlistName": "Summer Tour 2026", "songIdx": 7, "ts": 1711324800000 }
```

- Updated on every song change
- Expires after 24 hours
- `getResumeState()` returns state or null
- `clearResumeState()` on setlist completion
- Entry points can check and offer "Resume playback" (wiring is future)

## Edge Cases

- **YouTube API timeout:** 8s → fallback UI for every song
- **Cached ID for removed video:** Fallback shows, user can skip (cache doesn't auto-clear)
- **Resume state expired:** Returns null, no resume offered
- **iOS inline playback:** `playsinline: 1` prevents fullscreen takeover
- **Last song completes:** Resume state cleared, "Setlist complete" toast
