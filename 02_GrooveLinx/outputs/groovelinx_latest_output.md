_Generated by Claude · 2026-03-24_

## Summary

Added time-aware priority: items near upcoming rehearsals/gigs rise higher and get urgency tags. Lock In dashboard warns about rehearsal blockers. No new data structures — reads existing gig/calendar data.

## How Urgency Is Determined

`FeedActionState.getUrgencyTag(item, meta)` checks:

1. Item must be unresolved with tag `needs_input` or `mission_critical`
2. Compares against `_nextEventCache` (set by `setNextEvents()` during feed load)

| Condition | Tag | Tone |
|-----------|-----|------|
| Gig in ≤ 3 days | "Needed for upcoming gig" | Red |
| Rehearsal in ≤ 2 days | "Needed for next rehearsal" | Yellow |
| Gig in ≤ 7 days | "Needed for upcoming gig" | Yellow |
| No nearby event | No tag | — |

**Sort bias:** Within the same priority bucket, items with urgency tags sort higher (fewer days = higher priority). This is a tiebreaker, not a full reorder — critical items still come before needs-input items.

## Where UI Appears

### 1. Feed Cards — Urgency Tag
**Location:** Inline in card header, after status badge
**Visual:** Small tag with clock icon: "⏰ Needed for next rehearsal"
**Color:** Red (gig ≤ 3 days) or Yellow (rehearsal ≤ 2 days, gig ≤ 7 days)

### 2. Feed Sort Order
**Change:** Items near events rise within their priority group
**Example:** Two unvoted polls — one tagged "Needed for upcoming gig" sorts before the other

### 3. Lock In Dashboard — Rehearsal Blocker
**Location:** Inside Band Alignment card, below "X/Y decisions resolved"
**Trigger:** Rehearsal within 3 days AND unresolved items > 0
**Display:** "⚠️ 2 items still blocking rehearsal" (red text)

## Event Data Source

`_feedLoadNextEvents()` runs during feed data load:
- Reads `gigs` from `loadBandDataFromDrive('_band', 'gigs')` — finds next future gig
- Reads `calendar_events` — finds next future rehearsal
- Passes `{ rehearsal: "2026-03-28", gig: "2026-04-05" }` to `FAS.setNextEvents()`
- Dashboard reads via `FAS.getNextEvents()` for blocker warning

No new Firebase paths. No new data structures. Reuses existing data.

## Edge Cases

- **No upcoming events**: No urgency tags shown, sort unaffected, no blocker warning
- **Event today**: `daysUntil = 0`, treated as urgent (gig ≤ 3 or rehearsal ≤ 2)
- **Resolved items**: Never get urgency tags (checked first)
- **Archived items**: Never get urgency tags
- **FYI items**: Never get urgency tags (only `needs_input` and `mission_critical`)
- **Feed not loaded**: Dashboard blocker reads `FAS.getNextEvents()` which is null if feed hasn't loaded → no blocker shown (safe fallback)
