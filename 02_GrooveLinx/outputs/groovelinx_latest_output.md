## Failure States

### 1. App Not Configured (`SPOTIFY_CLIENT_ID` empty)

**Dialog:**
> **Spotify Sync Not Ready**
> Spotify sync is still being set up for GrooveLinx. Use Quick Listen for now.
> [▶ Quick Listen] [📺 YouTube] [Close]

- "Quick Listen" → opens first matched URL via `quickLaunch()`
- "YouTube" → opens first matched YouTube URL
- Technical detail only in `console.warn`

### 2. User Not Connected (no token)

**Dialog:**
> **Connect Spotify**
> Sync your GrooveLinx playlist to Spotify so you can listen in the car and download it there.
> [Connect Spotify] [Not now]

- "Connect Spotify" → starts PKCE OAuth flow

### 3. Session Expired (token expired, refresh failed)

**Dialog:**
> **Reconnect Spotify**
> Your Spotify session expired. Reconnect to keep syncing playlists.
> [Reconnect] [Cancel]

- "Reconnect" → starts PKCE OAuth flow
- Silent refresh attempted first — dialog only shows if refresh fails

## Detection Logic

`_getSpotifyFailureState()` returns:
- `'not_configured'` — no `SPOTIFY_CLIENT_ID`
- `'not_connected'` — no token data in localStorage
- `'expired'` — token data exists but `expiresAt` is past
- `'ok'` — valid token

Called at the top of `syncToSpotify()` before any API calls.

## Sync Flow (Updated)

```
syncToSpotify(bundleType)
  ├─ not_configured → dialog with Quick Listen / YouTube fallback
  ├─ not_connected → dialog with Connect button
  ├─ expired → try silent refresh
  │     ├─ refresh success → continue sync
  │     └─ refresh failed → dialog with Reconnect button
  └─ ok → proceed with sync
```

Also updated: mid-sync `getSpotifyUserId()` failure now shows the expired dialog instead of a toast.

## Edge Cases

- **Dialog dismissed:** No action taken, user can try again later
- **Quick Listen from dialog:** Falls through to `quickLaunch()` which opens first resolved URL
- **YouTube from dialog:** Falls through to `quickLaunch()` with YouTube destination
- **Multiple dialogs:** Previous dialog removed before showing new one (ID-based dedup)
