# GrooveLinx — Setlist Player v6.1: Speed + Confidence

_Generated by Claude · 2026-03-24_

## Changes from v6

### 1. Reduced Timeouts (8s → 4s max)

| Component | Before | After |
|-----------|--------|-------|
| YouTube per-backend | 4s | 2s |
| YouTube race total | 5s | 2.5s |
| Archive search | 4s | 2s |
| Chain total timeout | 8s | 4s |

Cached songs play **instantly** (0ms resolution).

### 2. Progressive Status Messages

Video container now shows which source is being tried:

- "Trying YouTube..." → "Trying Spotify..." → "Trying Archive..."
- Then: "Playing via YouTube · Best match"

Replaces static "Finding best version..." with real-time feedback.

### 3. Fallback Order Optimization

| Preference | Chain Order |
|-----------|------------|
| YouTube (default) | YouTube → Spotify → Archive |
| Spotify | Spotify → YouTube → Archive |
| Archive | Archive → Spotify → YouTube |

Spotify is always second (fastest API response). Archive always last (slowest).

### 4. Confidence Labels

Source label now shows match quality:

| Label | When |
|-------|------|
| **Best match** | Cached, Firebase-saved, or Spotify result |
| **Close match** | Secondary source fallback |
| **Live version** | Archive.org result |

Displayed as: "▶ Playing via YouTube · Best match"

### 5. Silent Retry

If the **first source** (preferred) fails on first attempt:
- Retries once silently before moving to next source
- Only for YouTube (where backend availability is unpredictable)
- User sees no difference — just "Trying YouTube..." stays slightly longer

### 6. Cache Quick Path

Before running the chain, checks ALL caches (not just preferred source):
1. Preferred source cache? → play instantly
2. Any cached YouTube ID + YT ready? → play instantly
3. Any cached Spotify track? → play instantly
4. No cache → run full chain

## Timing Flow

```
Cached song:     [instant] → Playing
YouTube hit:     [0-2.5s]  → Playing via YouTube · Best match
Spotify hit:     [0-1s]    → Playing via Spotify · Best match
Archive hit:     [0-2s]    → Playing via Archive · Live version
All fail:        [4s max]  → Last resort fallback UI
```
