## Practice Streak

`_getPracticeStreak()` counts consecutive days with practice_set, practice_all, or rehearsal actions. Looks back up to 30 days. Skips today if no practice yet (doesn't break the streak until tomorrow).

## Data Model (Scorecard-Ready)

All data needed for future scorecard is now structured:

### Per-session (Firebase: `rehearsal_sessions/{id}`)
- `rating`: great / solid / needs_work
- `summary`: auto-generated text
- `totalActualMin`, `totalBudgetMin`
- `songsWorked[]`
- `mixdown_id`, `mixdown_tag`
- `blocks[]`: per-song timing

### Per-user (localStorage: `gl_action_log`)
- Daily actions: practice_set, practice_all, rehearsal, completed_*
- 14-day rolling window

### Per-song (Firebase: readiness cache)
- Per-member readiness scores (1-5)
- Aggregated averages

### Band-level (computed)
- Rehearsal frequency (sessions/week)
- Session quality trend (rating averages)
- Active member count
- Weak song count
- Gig-ready percentage
