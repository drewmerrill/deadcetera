# GrooveLinx — Usage Instrumentation

_Generated by Claude · 2026-03-24_

## Summary

Lightweight usage tracking to identify real behavior gaps. Writes daily rollups to Firebase per member. No external analytics. Auto-flushes on visibility change.

## Metrics Design

### What's Tracked

| Metric | Event | Where Triggered |
|--------|-------|----------------|
| Feed visits | `feed_visit` | `renderBandFeedPage()` |
| Actions completed | `action_completed` | `_feedAdvanceOnboarding()` (vote, acknowledge, resolve) |
| Items created (quick) | `item_created {method:'quick'}` | `_feedQuickPost()` |
| Items created (structured) | `item_created {method:'structured'}` | `_feedSubmitCreate()` |
| Targeted creates | `item_created {targeted:true}` | When `targetType === 'specific'` |
| Time to first action | Computed | `firstActionAt - session.startedAt` |
| Bounce rate | Computed | Feed visits with 0 actions between them |

### Firebase Storage

**Path:** `bandPath('metrics/{memberKey}/{YYYY-MM-DD}')`

**Daily rollup shape:**
```json
{
  "visits": 5,
  "actions": 8,
  "creates": 2,
  "quickAdds": 1,
  "structuredAdds": 1,
  "targetedCreates": 0,
  "bounces": 1,
  "avgTimeToAction": 12,
  "_ttaCount": 3,
  "lastSeen": "2026-03-24T18:30:00Z"
}
```

### Session State (sessionStorage)

Tracks within-session counts, flushed to Firebase on:
- `visibilitychange` (user leaves/minimizes)
- Every 5 minutes while active

## How to View Results

**Console (quick check):**
```javascript
FeedMetrics.getWeekSummary().then(console.table)
```

Returns per-member 7-day totals:
```
{ drew: { visits: 12, actions: 28, creates: 5, bounces: 2, avgTimeToAction: 8, name: 'Drew' },
  chris: { visits: 3, actions: 1, creates: 0, bounces: 2, name: 'Chris' }, ... }
```

**Firebase Console:** Browse to `bands/{slug}/metrics/{memberKey}/` to see daily breakdowns.

**Key questions answered:**
- "Who's using the feed?" → `visits > 0` per member
- "Who creates vs only consumes?" → `creates > 0` vs `creates === 0`
- "Quick add vs structured?" → `quickAdds` vs `structuredAdds`
- "How fast do users act?" → `avgTimeToAction` (seconds)
- "Who bounces?" → `bounces / visits` ratio
- "Is targeting used?" → `targetedCreates` count

## Implementation — Zero Overhead

- No external libraries
- No network requests except daily rollup flush (single Firebase write)
- Session state in sessionStorage (no localStorage pollution)
- Graceful degradation: all tracking calls wrapped in `typeof FeedMetrics !== 'undefined'` checks
- Flush on `visibilitychange` ensures data survives tab close
