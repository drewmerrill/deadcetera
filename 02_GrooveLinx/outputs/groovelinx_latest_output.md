# GrooveLinx — Listening Bundles + Delivery Destinations

_Generated by Claude · 2026-03-24_

## Architecture Overview

**Bundle** (GrooveLinx-owned) = WHAT to listen to
**Destination** (provider-specific) = WHERE to listen

Bundles are computed from existing GrooveLinx data (setlists, readiness, practice attention, North Stars). Destinations handle delivery via existing URL resolution + openMusicLink.

## Data Model

### Bundle (computed, not stored)
```json
{
  "type": "gig",
  "title": "Next Gig",
  "songs": [
    { "songTitle": "Althea", "order": 0 },
    { "songTitle": "Sugaree", "order": 1 }
  ],
  "computed": "2026-03-24T..."
}
```

### Per-song version (existing, unchanged)
Stored at `spotify_versions` per song. Each version has:
- `url` — provider URL
- `platform` — 'spotify', 'youtube', 'archive', 'link'
- `isDefault` — band-voted default
- `votes` — member votes

### User Preferences (localStorage)
```json
{
  "defaultDestination": "spotify",
  "fallbackDestination": "youtube"
}
```

## Bundle Types

| Type | Source | Dashboard |
|------|--------|-----------|
| `gig` | Next gig → linked setlist → song list | Play |
| `rehearsal` | `GLStore.getPracticeAttention()` top 10 | Lock In |
| `focus` | Weak songs from readinessCache (avg < 3) | Sharpen |
| `northstar` | Songs with voted default North Star version | Future |

## Destination Behavior

| Destination | Phase | Behavior |
|-------------|-------|----------|
| Spotify | 1 (now) | Opens first matched Spotify URL via deep link / web fallback |
| YouTube | 2 | Same pattern — opens first matched YouTube URL |
| Archive.org | 3 | Opens first matched Archive URL |

**URL Resolution:** For each song in a bundle, `resolveUrl(songTitle, destination)` checks saved versions:
1. Default version matching destination platform
2. Any version matching destination platform
3. Default version (any platform)
4. First available version

## UI Entry Points

### Dashboard Listening Cards
Each mode dashboard has a listening card with 3 destination buttons:

- **Sharpen**: "Listen & Learn" → focus bundle (weakest songs)
- **Lock In**: "Rehearsal Prep" → rehearsal bundle (practice attention)
- **Play**: "Play Your Set" → gig bundle (next gig setlist)

Buttons: 🎵 Spotify (green) | 📺 YouTube (red) | 🏛️ Archive (neutral)

### Quick Launch Flow
1. User taps a destination button
2. `quickLaunch(bundleType, destination)` fires
3. Bundle computed from live data
4. URLs resolved per song for chosen destination
5. First URL opened via `openMusicLink()`
6. Toast: "12/15 matched in spotify"

## Edge Cases

- **No versions saved for a song**: Skipped in delivery, counted as `failed` in results
- **No setlist linked to next gig**: Gig bundle returns empty, toast "No songs found"
- **No active weak songs**: Focus bundle returns empty
- **ListeningBundles not loaded**: Listening card doesn't render (graceful absence)
- **Offline**: resolveUrl fails silently, toast shows 0 matched
