# GrooveLinx — Band Feed Cleanup Controls

_Generated by Claude · 2026-03-24_

## Problem
Noisy auto-posts ("Focus playlist updated — 1 songs") cluttering Band Feed/Room.

## Root Cause
`_postPlaylistToFeed()` in `listening-bundles.js` (line 1147) automatically posts to `bands/{slug}/ideas/posts` every time a playlist syncs. These system-generated posts have `_source: 'playlist_sync'` but were shown alongside user posts in the main feed.

## Changes

### 1. Single-Item Delete
Every feed card now has a 🗑️ delete button (visible to creator or admin).
- Requires confirmation dialog
- Permanently removes from Firebase (`ideas/posts/{id}` + `feed_meta/{key}`)
- Instant UI update (no reload)
- Toast: "Post deleted"

### 2. Permissions
`_feedCanDelete(item)` checks:
- `FeedActionState.isMe(item.author)` — creator can delete
- `getMyMemberKey() === 'drew'` — admin can delete
- Fallback: allow if identity can't be checked

### 3. Bulk Delete Mode
"☑ Select" button in filter bar toggles bulk mode:
- Checkboxes appear on every card
- Counter: "N selected"
- "🗑 Delete Selected" button with confirmation
- Toast: "5 items deleted"
- Cancel exits bulk mode

### 4. System Filter
New "⚙️ System" tab in filter bar shows only auto-generated posts (`_source === 'playlist_sync'`). Makes it easy to find and bulk-delete system clutter.

### 5. Auto-Post Suppression
System-generated posts now hidden from default "All" view. They only appear under the "System" filter tab.

`_postPlaylistToFeed()` is now **off by default** — only posts to feed if `localStorage.gl_playlist_feed_posts === 'true'`. This stops future clutter at the source.

### 6. System Badge
Posts with `_source: 'playlist_sync'` show a "SYSTEM" badge in the card, making them easy to identify.

## Files Changed

| File | What |

| `js/features/band-feed.js` | Delete action, bulk mode, System filter, system badge, auto-post hiding |
| `js/core/listening-bundles.js` | Suppressed auto-posts by default |

## Schema

No schema changes. Existing fields used:
- `item._source === 'playlist_sync'` — identifies system posts
- `item.author` — used for delete permission check
- Firebase paths: `ideas/posts/{id}` (delete source), `feed_meta/{key}` (delete metadata)
