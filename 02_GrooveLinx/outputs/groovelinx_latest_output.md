## Bug Fixes

### Spotify/Archive embeds never appeared
`_playSource()` for Spotify and Archive set state to PLAYING but never emitted `embedReady`. The UI's `_createEmbed` handler was never triggered. Fixed: both now emit `embedReady` with their data (trackId / identifier).

### Embed handler used stale engine state
`_createEmbed` went back to `GLPlayerEngine.getActiveResult()` instead of reading from the event data. Fixed: reads `d.trackId` / `d.identifier` directly.

## New Wiring

### Listen & Learn → Unified Engine
`hdPlayBundle(bundleType)`:
1. Computes bundle via `ListeningBundles.computeBundle()`
2. Extracts song titles
3. Loads queue into `GLPlayerEngine`
4. Shows overlay via `GLPlayerUI`
5. Falls back to legacy `quickLaunch` if engine unavailable

### Chart Highlighting
`songChange` event now calls `ChartSystem.highlightActiveSong(title)` — when viewing setlist charts while playing, the active song auto-highlights and scrolls into view.

## Player States

| State | Trigger | UI |
|-------|---------|-----|
| IDLE | No queue | Nothing |
| LOADING | `loadQueue()` / `loadFromSetlist()` | "Loading..." |
| RESOLVING | Song needs source lookup | "Trying YouTube..." / "Trying Spotify..." |
| PLAYING | Source embedded | Video/audio playing |
| FALLBACK | All sources failed (4s timeout) | Retry / Paste URL / Skip |
| ERROR | System error | Error + retry |

## Fallback Chain

### Default Mode
preferred → Spotify → YouTube → Archive

### Jam Mode
Archive → Spotify → YouTube

## Resolution Priority (per song)
1. Set-level override
2. Curation: override_id → north_star_id → selected_id
3. Firebase saved versions
4. Local cache
5. Auto-search (7 YouTube backends parallel, Spotify API, Archive API)
