## Filter Logic

### Tabs
`[ All (5) ] [ ✋ Need My Vote (2) ] [ ✅ Voted (3) ]`

### Classification
- **Need my vote**: `pending` pitches where `votes[myKey]` is null/undefined
- **Voted**: `pending` pitches where `votes[myKey]` has a value ('yes', 'no', 'defer')
- **All**: all `pending` pitches

### Auto-Default
- If `needVote.length > 0` → defaults to "Need My Vote"
- Otherwise → defaults to "All"
- Stored in `_spFilter` variable, persists within session

## Progress Indicator

Below filter tabs:

`[████████░░░░] 3/5 voted`

- Green at 100%, indigo otherwise
- Updates on re-render after each vote
- `pct = round(votedCount / totalCount * 100)`

## Empty States

| Filter | Condition | Display |
|--------|-----------|---------|
| Need My Vote | 0 remaining | ✅ "You're all caught up" / "No pitches need your vote right now." |
| Voted | 0 voted | "You haven't voted on any pitches yet." |
| All | 0 pitches + 0 backlog | "No active pitches. Suggest a song for the band to learn." |

## UI Rules

- Filter tabs are always visible when pending pitches exist
- No dropdowns, no extra clicks
- Segmented control style (same as Feed filter pattern)
- Progress bar same visual language as Feed progress bar

## Edge Cases

- **No member key**: `myKey = null` → all pitches show as "need vote" (safe fallback)
- **Vote then re-render**: Filter refreshes via `_spSetFilter()` which re-calls `renderSongPitchSection()`
- **Re-pitch from backlog**: Doesn't interact with filter (backlog section is separate)
- **Recently decided section**: Unaffected by filter (always visible at bottom)
