## Resume UX

**Before:** 3-second delay after DOMContentLoaded
**After:** `requestAnimationFrame` — shows as soon as DOM is ready (effectively instant)

**Visual upgrade:**
- Gradient background, stronger border, larger shadow
- Play icon in a circle on the left
- Setlist name in bold: "Resume Summer Tour 2026?"
- Song name in bold: "Continue from **Sugaree** · 7/15"
- Larger "Resume" button (10px/20px padding)
- Fade-up animation on appearance

## Now-Playing Bar

**Before:** Play icon + song info + close button
**After:** Play/Pause button + song info + Skip button + close button

```
[⏸] Sugaree         [⏭] [✕]
     7/15 · Tap to return
```

- **Play/Pause (⏸/▶):** Toggles YouTube player without opening overlay
- **Song info area:** Tappable — opens full player overlay
- **Skip (⏭):** Advances to next song in queue
- **Close (✕):** `fullClose()` — destroys everything

## Completion Message

**Before:** "Setlist complete"
**After:** "🎶 Set complete — nice run"

Feels like positive reinforcement, not just a status update.

## Edge Cases

- **Resume prompt + now-playing bar:** Resume prompt has lower z-index (8500) than now-playing bar (9500). If both exist, bar takes visual priority.
- **Play/Pause on now-playing bar:** Works even when YouTube player is in overlay (player instance persists across overlay open/close)
- **Skip from bar:** Advances song, updates bar (via `_playCurrent` → `_showNowPlayingBar` on next close)
