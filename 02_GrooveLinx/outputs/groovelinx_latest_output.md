# GrooveLinx — Spotify Config Auto-Fetch

_Generated by Claude · 2026-03-24_

## Summary

Spotify Client ID now fetched automatically from Cloudflare Worker at runtime. No hardcoded secrets. Normal users never see "not configured" if the worker has the secret set.

## Files Changed

| File | What |
|------|------|
| `js/core/listening-bundles.js` | `_ensureSpotifyConfig()` fetches Client ID from worker, called before sync/connect |
| `worker.js` | New `GET /spotify-config` endpoint returns `{ clientId }` from env secrets |

## How It Works

1. User taps "Sync to Spotify"
2. `syncToSpotify()` calls `_ensureSpotifyConfig()` (once per session)
3. Fetches `GET /spotify-config` from Cloudflare Worker (3s timeout)
4. Worker returns `{ clientId: env.SPOTIFY_CLIENT_ID }` from its stored secrets
5. Client stores in `SPOTIFY_CLIENT_ID` variable — used for all subsequent OAuth calls
6. If worker is unreachable or secret not set → falls back to "not configured" dialog

## Remaining Manual Steps

### 1. Add Redirect URI in Spotify Dashboard

Go to **https://developer.spotify.com/dashboard** → your app → Settings → Redirect URIs

Add: **`https://app.groovelinx.com/`** (exact, with trailing slash)

This is the ONLY manual step. The Client ID + Secret are already in Cloudflare as secrets. The code now reads them automatically.

### 2. Deploy Worker

The worker needs to be redeployed with the new `/spotify-config` endpoint:
```bash
wrangler deploy
```

## After Setup

| User sees | When |
|-----------|------|
| "Connect Spotify" | First use (app configured, user not connected) |
| "Reconnect Spotify" | Token expired, refresh failed |
| Successful sync | Normal operation |
| "Spotify Sync Not Ready" | Only if worker secret is missing or worker unreachable |
