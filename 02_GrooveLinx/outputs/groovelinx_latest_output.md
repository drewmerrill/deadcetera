# GrooveLinx — Progression + Completion Tracking

_Generated by Claude · 2026-03-24_

## Action Completion Tracking

Lightweight localStorage tracker (`gl_action_log`):

| Action | When Logged |
|--------|------------|
| `practice_set` | User taps "Practice Set" or runs gig bundle |
| `practice_all` | User taps "Practice All" (focus bundle) |
| `completed_set` | Queue finishes (queueEnd event) |
| `completed_all` | Focus queue finishes |

Keeps last 14 days. No Firebase writes — instant, local.

## Next Action Card: Completion-Aware

| Already practiced today? | Card shows |
|--------------------------|-----------|
| No, weak songs exist | "5 songs need work" → Start Practicing |
| No, all songs good | "You're in good shape" → Practice Set |
| Yes, weak songs remain | "✅ Practiced today — 3 songs still need attention" → Keep Going |
| Yes, all good | "✅ Practiced today — you're locked in" → Run the Set |

Card turns green when completed. Label changes from "Next Up" to "Done Today".

## Progression Signals

Chips below the Next Action card:

- "🎵 3 practice sessions this week"
- "↑ Last 5 rehearsals trending up"
- "→ Rehearsals holding steady"
- "🔒 All songs in good shape"

Derived from action log + rehearsal session ratings + readiness cache.

## Player Context

When Practice All launches, the overlay header shows:

Context strings per bundle type:

Reinforces purpose during playback.

## The Loop

```
Action → Practice → Improvement → Next Action
  ↑                                    ↓
  └────────────────────────────────────┘
```

1. User sees "5 songs need work" → **Start Practicing**
2. Player opens: "Focus Set · Practicing weakest songs (3)"
3. User practices through the set
4. Queue ends → action logged
5. User returns to home → "✅ Practiced today — 3 songs still need attention" → **Keep Going**
6. Progression signal: "🎵 2 practice sessions this week"
7. Next day: readiness improved → "3 songs need work" (was 5)
