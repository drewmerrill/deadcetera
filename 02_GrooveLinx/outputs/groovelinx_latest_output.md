## Spotify Auth Flow (PKCE — no server)

1. User taps "Sync to Spotify"
2. If not connected: PKCE OAuth redirect to Spotify
3. User grants permission on Spotify
4. Redirects back to GrooveLinx with auth code
5. `handleSpotifyCallback()` exchanges code for token (client-side)
6. Token stored in `localStorage('gl_spotify_token')` with expiry
7. Subsequent taps use stored token

**Setup required:** Drew must register a Spotify App at developer.spotify.com:
- Set redirect URI to `https://app.groovelinx.com/`
- Copy Client ID into `SPOTIFY_CLIENT_ID` in listening-bundles.js

## Playlist Sync Logic

`syncToSpotify(bundleType)`:

1. **Compute bundle** — songs from gig setlist, rehearsal attention, or focus weak songs
2. **Resolve track IDs** — for each song:
   - Check saved versions for Spotify URLs → extract track ID
   - Fallback: search Spotify API for "songTitle Grateful Dead"
3. **Create or reuse playlist** — persistent per bundle type:
   - `GrooveLinx — Upcoming Gig`
   - `GrooveLinx — Rehearsal Prep`
   - `GrooveLinx — Focus Songs`
   - Playlist IDs stored in `localStorage('gl_spotify_playlists')`
4. **Replace contents** — PUT to `/playlists/{id}/tracks` with resolved URIs
5. **Open playlist** — via Spotify deep link (app) or web URL

## Edge Cases

- **Not connected**: "Sync to Spotify" triggers OAuth flow, returns after auth
- **Token expired**: Auto-detected, user prompted to reconnect
- **No Client ID**: Shows toast "Spotify app not configured yet"
- **No matches**: Shows "No Spotify matches found" (uses search API as fallback)
- **Empty bundle**: Shows "No songs in [type]"
- **Playlist already exists**: Reuses same playlist ID, replaces contents (order preserved)
