# GrooveLinx — Spotify Web Playback SDK Subsystem

_Generated by Claude · 2026-03-25_

## Architecture

```
GLPlayerEngine._playSpotify(trackId)
  ↓
  Try GLSpotifyPlayer.playTrackId(trackId)
  ├── SDK available + Premium → PLAYING (full track, no iframe)
  ├── Requires interaction (iOS) → "Tap play to start"
  └── SDK unavailable → Fallback to Spotify embed iframe
```

## New File: `js/core/gl-spotify-player.js`

Dedicated Spotify Web Playback SDK subsystem.

### State Machine

```
IDLE → LOADING_SDK → CONNECTING → READY → PLAYING / PAUSED
                                        → REQUIRES_INTERACTION
                                        → ERROR
                                        → UNAVAILABLE
```

### Capabilities
- Load Spotify Web Playback SDK (`sdk.scdn.co/spotify-player.js`)
- Create browser player device ("GrooveLinx")
- Transfer playback + start track
- Pause / Resume / Toggle / Seek / Volume
- Token refresh (auto-refresh on expiry, retry on 401)
- Track metadata from SDK state events

### User-Facing Messages

| State | Message |
|-------|---------|
| LOADING_SDK | "Loading Spotify..." |
| CONNECTING | "Connecting to Spotify..." |
| READY | "Spotify player ready" |
| PLAYING | "Playing in Spotify" |
| PAUSED | "Paused" |
| REQUIRES_INTERACTION | "Tap play to start Spotify on this device" |
| ERROR | Specific error message |
| UNAVAILABLE | "Spotify unavailable — using fallback" |

## Auth Scope Changes

**Before:**
```
user-read-email user-read-private playlist-modify-public playlist-modify-private
```

**After:**
```
user-read-email user-read-private playlist-modify-public playlist-modify-private streaming user-read-playback-state user-modify-playback-state
```

Added: `streaming`, `user-read-playback-state`, `user-modify-playback-state`

**Note:** Users who connected Spotify before this change will need to re-authorize to get the new scopes. The SDK will fail with an auth error, and the system falls back to embed gracefully.

## Integration with GLPlayerEngine

`_playSpotify(trackId)` in gl-player-engine.js:
1. Check if `GLSpotifyPlayer` is available and not UNAVAILABLE
2. Try `GLSpotifyPlayer.playTrackId(trackId)`
3. If SDK succeeds → emit `embedReady` with `source: 'spotify_sdk'`
4. If requires interaction → emit with `source: 'spotify_sdk_interaction'`
5. If unavailable → fall back to iframe embed (`source: 'spotify'`)

## Desktop vs Mobile

| Platform | Behavior |
|----------|----------|
| **Desktop Chrome/Firefox** | SDK loads, creates device, full-track playback |
| **Desktop Safari** | SDK loads, may require click to start |
| **iOS Safari** | SDK loads but autoplay blocked → REQUIRES_INTERACTION → tap CTA |
| **iOS in-app browser** | SDK may not load → UNAVAILABLE → embed fallback |
| **Android Chrome** | SDK loads, may auto-start or require tap |
| **Non-Premium user** | UNAVAILABLE ("Spotify Premium required") → embed fallback |

## Error Handling

| Error | Handling |
|-------|----------|
| SDK fails to load | → UNAVAILABLE, fallback to embed |
| SDK load timeout (10s) | → UNAVAILABLE, fallback to embed |
| Premium required | → UNAVAILABLE with clear message |
| Token expired | → Auto-refresh, retry once, then ERROR |
| Connection timeout (8s) | → ERROR, fallback to embed |
| 403 on play | → UNAVAILABLE ("Premium required") |
| 404 on play | → REQUIRES_INTERACTION |
| Network error | → ERROR with message |

**Every failure ends in**: clear message + fallback path. Never stuck.
