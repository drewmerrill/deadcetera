## Fix 1 — Spotify State (4 states)

| State | Label | Style |
|-------|-------|-------|
| `disconnected` | "🎵 Connect Spotify" | Green prominent |
| `connected_no_playlist` | "🎵 Create Playlist" | Green prominent |
| `synced` | "✅ Synced to Spotify" | Green dim |
| `stale` | "🔄 Update Playlist" | Yellow/amber |

Staleness detected by comparing `gl_sync_hash_{type}` vs `gl_current_bundle_hash_{type}` in sessionStorage. If hashes differ, playlist is stale.

## Fix 2 — Help System (object storage)

**Storage:** `gl_help_flags` → `{ surface: { seen: bool, disabled: bool } }`

| Action | Effect |
|--------|--------|
| Auto-trigger | Only if `!seen && !disabled` |
| "Got it" (`dismissHelp`) | `seen=true`, replay still allowed |
| "Don't show again" (`disableHelp`) | `seen=true, disabled=true`, no replay |
| "?" replay | Works if `!disabled` (via `canReplayHelp`) |

## Fix 3 — Interaction Lock

```javascript
function _isLocked() {
    return _state === STATES.PREPARING || _state === STATES.STARTING;
}
```

All three entry points (`preparePlaybackBundle`, `launchPlaybackOption`, `advancePlaybackBundle`) check `_isLocked()` first. Prevents double-trigger.

## Fix 4 — Actionable Error Messages

| Scenario | Before | After |
|----------|--------|-------|
| No URL | "No playable URL for this track" | "No playable URL — tap below to search manually" |
| Popup blocked | "Couldn't start playback" | "Popup blocked — tap the link below to open" + shows fallback overlay |
| No sources | "No playable sources found" | same (already clear) |

On popup block: catches the error, shows fallback overlay with manual links instead of just a toast.

## Fix 5 — Bar State Cleanup

`_removeBar()` now transitions state from PLAYING back to IDLE when user dismisses the now-playing bar. Prevents stale PLAYING state.
