## Scoring Rules

### Health Summary (headline answer)

| Trend | Readiness | Summary | Color |
|-------|-----------|---------|-------|
| Improving + high ready | >50% gig-ready | "Band is trending up and nearing gig-ready" | Green |
| Improving | Any | "Momentum building — keep rehearsing" | Green |
| Steady + high ready | >60% gig-ready | "Holding strong — band is in good shape" | Indigo |
| Steady | Any | "Steady but not growing — push harder" | Gray |
| Declining | Any | "Needs attention — recent sessions slipping" | Amber |
| No sessions + some ready | >50% | "Songs in decent shape — start tracking rehearsals" | Gray |
| Default | | "Getting started — keep practicing and rehearsing" | Dark gray |

### Trend Analysis

Compares average rating of recent half vs older half of last 5 sessions:
- Recent > Older + 0.3 → **Improving**
- Recent < Older - 0.3 → **Declining**
- Otherwise → **Steady**

### Strengths (auto-detected)

| Signal | Condition |
|--------|-----------|
| "Consistent quality rehearsals" | Average rating ≥ 2.5 |
| "Good time management" | 3+ of last 5 sessions on time (±3 min) |
| "Active rehearsal schedule" | 2+ sessions this week |
| "N% of songs gig-ready" | ≥80% of active songs avg ≥ 4.0 |

### Issues (auto-detected)

| Signal | Condition |
|--------|-----------|
| "Sessions running long" | 2+ of last 5 over by >10 min |
| "No rehearsals this week" | 0 sessions in last 7 days |
| "N songs below readiness threshold" | >3 songs with avg ≤ 2.0 |

### Song Movement

| Category | Definition |
|----------|-----------|
| Improved | Songs with avg readiness ≥ 4.0 |
| Declining | Songs with avg readiness ≤ 2.0 |
| Steady | Everything in between |

## Data Sources

| Data | Source | Path |
|------|--------|------|
| Session ratings | Firebase | `rehearsal_sessions/{id}.rating` |
| Session timing | Firebase | `rehearsal_sessions/{id}.totalActualMin/totalBudgetMin` |
| Readiness scores | Firebase | `readinessCache` (per-song per-member) |
| Song status | GLStore | `getStatus(title)` |
| Practice frequency | localStorage | `gl_action_log` |

## Integration

Scorecard appears on:
- **Sharpen** dashboard (below Top Songs)
- **Lock In** dashboard (below Next Action, above alignment)
- **Play** dashboard (future)

Returns empty string if no data — no empty card.
