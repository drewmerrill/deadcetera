## Search Ranking Logic

`_pickBestResult(results, songTitle, bandName)` scores up to 8 results:

| Signal | Score |
|--------|-------|
| Artist name in video title or author | +50 |
| Song title in video title | +30 |
| Duration 3-15 minutes (typical song) | +20 |
| Duration > 15 minutes (long jam) | +10 |
| Views > 1M | +15 |
| Views > 100K | +10 |
| Views > 10K | +5 |
| Search position bonus | +5 to +1 |

**Filtered out:**
- Duration < 2 minutes (clips)
- Titles containing: cover, karaoke, tutorial, lesson, reaction, parody

**Fallback:** If all results are filtered out, takes the first result anyway.

## User Override — "Use this version"

**Location:** Bottom-right of player overlay, next to "Up next"
**Label:** "🔒 Use this version"

**On click:**
1. Saves current YouTube video as a locked version to Firebase (`spotify_versions`)
2. Updates localStorage cache
3. Button changes to "✅ Locked" (green, disabled)
4. Toast: "🔒 Version locked for Althea"

**Data saved:**
```json
{
  "id": "yt_locked_1711324800000",
  "url": "https://www.youtube.com/watch?v=abc123",
  "platform": "youtube",
  "title": "YouTube (locked)",
  "addedBy": "drewmerrill1029@gmail.com",
  "dateAdded": "2026-03-24",
  "notes": "Locked from Play Mode"
}
```

## Edge Cases

- **All results filtered:** Falls back to first result (better than nothing)
- **Lock with no video playing:** "No video playing to lock" toast
- **Lock fails (Firebase):** "Could not save" toast
- **Already locked version:** Lock button still works (adds another locked version, but cache takes priority)
