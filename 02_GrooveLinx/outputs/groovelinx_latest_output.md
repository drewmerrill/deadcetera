## Progress Bar

**Location:** Between page header and attention bar
**Display:** `{completed}/{total}` with filled bar
**Calculation:**
- `total` = `needsMyInput` count at session start (captured on first feed load)
- `completed` = `total - currentNeedsMyInput`
- `pct` = `completed / total * 100`

**Behavior:**
- Hidden if total <= 1 (no progress to show for single item)
- Bar fills with indigo gradient, smooth 0.4s CSS transition
- Count turns green at 100%
- Resets on new feed load (`_feedSessionTotal = 0`)
- Updates instantly after every action

## Auto-Advance

After each action completes:
1. Feed re-renders (items re-sort by priority)
2. 350ms later, `_feedAutoAdvance()` runs
3. Finds first remaining `needsMyInput` item in the cache
4. Applies `feed-next-action` class (subtle indigo border + background tint)
5. Scrolls into view if off-screen (`behavior: smooth, block: center`)
6. Previous auto-advance highlight removed before applying new one

**Visual:** Lighter than first-action pulse ‚Äî no animation, just a subtle border/background shift. Disappears when that item is completed or feed re-renders.

## Momentum Nudges

After EACH action (not just the first):

| Remaining | Message |
|-----------|---------|
| > 2 | "{N} left ‚Äî keep going" |
| 2 | "Two left ‚Äî almost there" |
| 1 | "Last one ‚Äî finish it" |
| 0 | (handled by completion celebration) |

Delivered via lightweight toast (2s, fades out).

## Completion Celebration

When `needsMyInput === 0` after a rerender:
- Only triggers if `_feedSessionTotal > 1` (user actually completed multiple items)
- Applies `feed-completion-celebrate` class to the all-clear banner
- CSS: subtle scale animation (1.0 ‚Üí 1.015 ‚Üí 1.0 over 0.6s)
- Class removed after 2s
- Combined with the existing "You're locked in" banner content

## Edge Cases

- **Single action item**: Progress bar hidden (total <= 1), nudge still fires
- **New feed load**: Session total resets, progress starts fresh
- **Action via deep link + return**: Progress resumes from where cache left off
- **Auto-advance with no remaining items**: No highlight applied, completion fires instead
- **User on filtered view**: Auto-advance only highlights items visible in current filter (finds by ID in DOM)
