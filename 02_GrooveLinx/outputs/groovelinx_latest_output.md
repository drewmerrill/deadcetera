# GrooveLinx — Single Primary Version + UI Refinement

_Generated by Claude · 2026-03-24_

## Summary

Each song now has one primary version. Lock button moved under song title. Resolution priority enforced. Toast controlled.

## Files Changed

| File | What |
|------|------|
| `js/features/setlist-player.js` | Primary version enforcement, UI placement, toast control, resolve priority |

## Data Model

When user clicks "Use this version":

**Before:** Appended a new locked version (could accumulate duplicates)
**After:** Removes any existing primary/locked YouTube versions, then saves new one:
```json
{
  "id": "yt_primary_1711324800000",
  "url": "https://www.youtube.com/watch?v=abc123",
  "platform": "youtube",
  "title": "YouTube (primary)",
  "isPrimary": true,
  "addedBy": "drewmerrill1029@gmail.com",
  "notes": "Set from Play Mode"
}
```

Only ONE primary YouTube version per song at any time.

## Playback Priority

`_resolveYouTubeId()` now resolves in strict order:
1. **Local cache** (`gl_yt_id_cache`) — instant
2. **Primary locked version** (`isPrimary: true, platform: 'youtube'`) — highest saved priority
3. **Any YouTube version** — non-primary saved versions
4. **Default North Star** — extract YouTube ID from default version URL
5. **Auto-search via Invidious** — ranked results

## UI Change

**Lock button moved:** From bottom-right (next to "Up next") to directly below song title/artist, centered. Clearly tied to the current song.

```
    Song Title
    Artist
  [🔒 Use this version]
    3 of 15
```

After click: button changes to "✅ Locked" (green). Toast: "🔒 Primary version set for Althea"

## Toast Control

"✅ Found a great version" now fires only once per song per session (sessionStorage key `gl_resolved_{title}`). Repeated plays of the same song don't re-toast.

## Edge Cases

- **Existing primary overwritten:** Old primary filtered out, new one saved. Cache updated.
- **Multiple platforms:** Primary only enforced for YouTube. Spotify locked versions (from match review) are separate.
- **No video playing:** "No video playing to lock" toast if button clicked during fallback state.
