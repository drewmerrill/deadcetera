Full-screen setlist player that plays through songs in order using YouTube embeds. Entry points on both setlist and gig views. Auto-advances between songs. Falls back to Spotify/Archive links.

## Player Architecture

### Entry Points
- **Gig view:** "▶️ Play Setlist" button next to "Go Live"
- **Setlist list:** 🎧 button on each setlist card

### Playback Logic

For each song in setlist order:

1. **Check saved versions** for YouTube URL → extract video ID
2. **Check default North Star** version → extract YouTube ID if YouTube URL
3. **If found:** Play via YouTube IFrame API (embedded, with controls)
4. **If not found:** Show fallback with links to YouTube search, Archive.org search, and Spotify

### Auto-Advance
- YouTube `onStateChange` detects `ENDED` state
- Automatically loads next song's video
- On last song: "Setlist complete" toast

### Controls
- **Play/Pause:** Toggles YouTube player state
- **Next:** Advances to next song
- **Previous:** Goes back one song
- **Close (✕):** Destroys player and overlay

### Fallback UI (no YouTube video found)
Shows "No video available" in player area + three action buttons:
- 📺 YouTube → opens YouTube search for the song
- 🏛️ Archive → opens Archive.org search
- 🎵 Spotify → resolves URL via ListeningBundles, opens in Spotify

Plus: "Skip to next ⏭" button

## Playback Engine

Uses **YouTube IFrame API** (loaded on demand):
- `_ensureYouTubeAPI()` injects the script tag, waits for `onYouTubeIframeAPIReady`
- Creates `YT.Player` with autoplay, modest branding, no related videos
- Reuses player instance across songs via `loadVideoById()`
- On error: shows fallback UI instead of crashing

## Edge Cases

- **YouTube API fails to load:** 8-second timeout, falls back to showing links for every song
- **No saved versions for any song:** Every song shows fallback UI with search links
- **Empty setlist:** "No songs in setlist" toast, player doesn't open
- **Last song ends:** "Setlist complete" toast, player stays open (user closes manually)
- **Video error (removed/blocked):** Auto-shows fallback with skip option
- **Player already open:** `close()` called before creating new overlay
- **Mobile:** Full-screen layout works on portrait and landscape
