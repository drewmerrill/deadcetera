# GrooveLinx — Push Notifications

_Generated by Claude · 2026-03-24_

## Architecture Overview

Two notification layers:

**Layer 1 — Local (works NOW):** Firebase real-time listener fires browser Notification API when app is backgrounded. No server needed. Triggers on new polls/ideas added to Firebase.

**Layer 2 — Push (ready, needs Cloud Function deployment):** VAPID subscription stored in Firebase per user. Service worker handles push events. Cloud Function code needed to send pushes when app is fully closed.

## Subscription Flow

1. User opens Settings → Notifications → taps "Off" → "On"
2. `enablePush()` calls `Notification.requestPermission()`
3. If granted: subscribes via `pushManager.subscribe()` with VAPID public key
4. Subscription stored in Firebase at `bandPath('push_subscriptions/{memberKey}')`
5. Also stored in `localStorage('gl_push_subscription')`
6. `gl_push_enabled = '1'` set

**Unsubscribe:** `disablePush()` unsubscribes from push manager, removes Firebase subscription, sets `gl_push_enabled = '0'`

## Trigger Logic

**Real-time triggers (local notifications):**
- New poll added (child_added on `polls`) where:
  - Item is < 60 seconds old (avoids firing on initial load)
  - Current user has NOT voted
  - Current user is NOT the author
  - → Fires: "Band needs your input" + question text

- New idea added (child_added on `ideas/posts`) where:
  - Item is < 60 seconds old
  - Tag is `needs_input` or `mission_critical` (NOT fyi)
  - Current user is NOT the author
  - → Fires: "New idea shared" + title text

**Suppression:**
- Only fires when `document.visibilityState !== 'visible'` (app backgrounded)
- Respects `gl_notif_prefs` (action_required ON by default, band_updates OFF)
- Never fires for FYI items

## Click Behavior

1. User taps notification
2. Service worker `notificationclick` handler fires
3. Finds existing app window → focuses it
4. Posts `GL_NOTIF_TAP` message with `{itemType, itemId}`
5. band-feed.js message handler navigates to feed
6. After 1.5s delay (feed load): scrolls to item, highlights with indigo border (3s)

If no existing window: opens `/#feed`

## Settings UI

**Location:** Settings → 🔔 Notifications tab

**Controls:**
- Master toggle: On/Off (triggers permission request on first enable)
- Action required: On/Off (polls, decisions needing input)
- Rehearsal & gig changes: On/Off (urgent near-event changes)
- Band updates: On/Off (general activity — OFF by default)

**Edge cases:**
- Notifications unsupported → red warning message
- Notifications denied → instructions to update browser settings
- Toggle off → unsubscribes + removes Firebase subscription

2. Open GrooveLinx on iPad, minimize to home screen
3. From another device, create a poll in Band Room
4. iPad shows notification: "Band needs your input: [question]"

6. Settings → Notifications → toggle Action required Off → no more poll notifications
