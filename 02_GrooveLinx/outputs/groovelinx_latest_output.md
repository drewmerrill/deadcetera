## Trigger Conditions

- **Auto-run:** First-ever feed visit (checks `localStorage('gl_wt_feed-walkthrough-v1')`)
- **Timing:** 1.2s delay after feed renders (gives DOM time to populate)
- **Skip:** Available on every step
- **Dismiss:** Escape key, Skip button, or completing all steps
- **Completion:** Marked done in localStorage, never auto-runs again

## How to Replay

- **"?" button** in feed header now launches the walkthrough (instead of static help banner)
- Calls `glSpotlight.run('feed-walkthrough-v1', null, { force: true })` which bypasses the completion check
- Fallback: if spotlight system not loaded, shows static banner

## Implementation Notes

- Uses existing `gl-spotlight.js` system (overlay + positioned dialog + glow ring)
- Steps use function targets for dynamic elements ("You owe this" may not exist on every load)
- Missing targets auto-skip to next step (graceful degradation)
- `_feedRegisterWalkthrough()` runs on file load
- `_feedTriggerWalkthrough()` runs after `renderBandFeedPage` completes

## Edge Cases

- **No feed items:** Steps 2, 3, 5 may not find targets → auto-skip to next step
- **Spotlight not loaded:** Walkthrough silently doesn't run
- **Help button without spotlight:** Falls back to static banner
- **Repeat visits:** Never auto-runs after first completion
- **Force replay:** "?" button always works regardless of completion state
