## Summary

Added session summary on completion, new-items indicator on return, and light weekly action history. Users feel that completing actions matters and develop return habits.

## Files Changed

| File | What |
|------|------|
| `js/features/band-feed.js` | Session summary, new-items indicator, action history tracking, continuity signals |

## Session Summary

**Trigger:** `needsMyInput === 0` AND `_feedSessionCompleted > 1`
**Display:** Inline below completion banner, centered text:
> "You cleared 4 items."
> "Band is tighter because of this."

**Behavior:**
- Appears once per completion (guarded by element ID)
- Auto-fades after 8 seconds
- Does not persist or reappear on re-render

## New-Items Indicator

**Detection:** On feed load, compares current `needsMyInput` against `localStorage('gl_feed_last_count')`
**Trigger:** Previous session ended at 0, current count > 0
**Display:** Indigo banner above attention bar:
> "3 new things need you"

**Behavior:**
- One-time per session (auto-fades after 6 seconds)
- `gl_feed_last_count` updated on every feed load

## Action History

**Storage:** `localStorage('gl_feed_action_history')` — JSON array:
```json
[{ "date": "2026-03-24", "count": 5 }, { "date": "2026-03-23", "count": 3 }]
```

**Rolling window:** Last 7 days only (older entries pruned on each write)
**Tracking:** `_feedRecordAction()` called on every `_feedAdvanceOnboarding()` — increments today's count

**Display:** Shown intermittently (~1 in 3 feed visits) when weekly count >= 3:
> "You've completed 12 actions this week"

Same auto-fade (6s) as new-items indicator.

## How Counts Stay Consistent

| Surface | Source |
|---------|--------|
| Progress bar | `_feedSessionTotal - computeSummary().needsMyInput` |
| Session summary | `_feedSessionCompleted` (incremented on each action) |
| New-items indicator | `computeSummary().needsMyInput` vs `gl_feed_last_count` |
| Weekly history | `gl_feed_action_history` rolling sum |
| Nav badge | `computeSummary().needsMyInput` via `setActionCount()` |

All use `FeedActionState.computeSummary()` for the live count. History uses its own lightweight counter that increments in sync.

## Edge Cases

- **First-ever session**: `gl_feed_last_count` is `-1` (not set) → new-items indicator skipped (only fires when previous was explicitly 0)
- **Single action completed**: Session summary skipped (requires > 1 to feel meaningful)
- **Weekly signal**: Only shows when count >= 3 and only ~33% of sessions (avoids repetition)
- **Session total resets on new feed load**: Returning to feed starts fresh progress tracking
- **History pruning**: Old entries cleaned on each action write, never grows unbounded
