# GrooveLinx — Setlist Player v6: Auto-Fallback Chain

_Generated by Claude · 2026-03-24_

## What Changed

Setlist player v5 → v6. Goal: tap → music → next. No dead ends.

## 1. Source Preference

New setting: preferred source (YouTube / Spotify / Archive).

- Stored in `localStorage` as `gl_player_source_pref`
- Configurable in Settings > Band > Playback Preference
- Also changeable live in player overlay (dropdown in header)
- Default: YouTube

Playback order: preferred source first, then the other two automatically.

## 2. Auto-Fallback Chain

When a song plays, the chain runs automatically:

```
_resolveSong(song):
  for each source in [preferred, fallback1, fallback2]:
    result = trySource(source, song)
    if found → embed in-app → done
  if ALL fail → show last-resort fallback
```

### Source resolvers:

| Source | Resolution Steps |
|--------|-----------------|
| YouTube | 1. localStorage cache → 2. Firebase saved versions → 3. Race 7 backends (5 Invidious + 2 Piped) |
| Spotify | 1. localStorage cache → 2. Firebase saved versions → 3. Spotify Search API (requires auth) |
| Archive | 1. Archive.org Advanced Search API → first audio result |

All three are tried automatically. User sees "Finding best version..." and then the embedded player appears.

### Chain timeout: 8 seconds total. If exceeded → last-resort fallback.

## 3. Simplified Fallback UI

Only shown when ALL three sources fail. Minimal design:

**Primary**: Auto-attempts Spotify search + embed as last resort.

**"More options" expander** (hidden by default):
- Paste YouTube URL → embed
- Retry all sources button

**Skip button** always visible.

No multi-button platform chooser by default. Just: searching → found → playing.

## 4. Spotify UX

When Spotify embed loads:
- Source label: "Playing via Spotify"
- Helper text: "Tap play in Spotify to start"
- No autoplay assumed (Spotify requires user interaction)

## 5. Always-Works Guarantee

Every song produces playable audio through this chain:
1. YouTube embed (auto-resolved) — covers most songs
2. Spotify embed (track search) — covers anything on Spotify
3. Archive embed (audio search) — covers live recordings
4. Last resort: Spotify search as final auto-attempt
5. Manual: paste YouTube URL (in "More options")
6. Skip: advance to next song

No dead ends. No silent no-ops. No external redirects by default.

## 6. Player Shell Behavior

- Source label shows which platform is playing ("Playing via YouTube", etc.)
- Switching source preference re-resolves current song immediately
- Setlist name, song title, controls, up-next bar all preserved
- Source switch doesn't reset queue position
- Spotify track IDs cached in localStorage (`gl_sp_track_cache`)

## Architecture

```
launch()
  → fullClose() + reset
  → _buildQueue() — synchronous, cache-only
  → _createOverlay() — with source pref dropdown
  → _playCurrent()

_playCurrent()
  → quick path: cached ID for preferred source? → embed immediately
  → else: _resolveAndPlay()

_resolveAndPlay()
  → 8s timeout
  → _resolveSong() — tries chain: [pref, fallback1, fallback2]
  → result? → _embedYouTube / _embedSpotify / _embedArchive
  → all fail? → _showLastResortFallback()

_showLastResortFallback()
  → auto-attempts Spotify search + embed
  → "More options": paste URL, retry
  → Skip button
```
