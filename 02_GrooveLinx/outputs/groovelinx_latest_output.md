## Summary

Two features shipped together: (1) Time + urgency layer with event-aware priority, and (2) Feed navigation + creation loop making Band Feed a true central hub.

## Feed Navigation

**Home → Feed:** Action Owed card is fully clickable (entire card, not just button). Arrow indicator added. Removed redundant "Open Feed →" button.

**Feed → Deep Link:** Back bar persists via sessionStorage. Restores filter + scroll position on return.

## Creation Entry Point

**Location:** Top of Band Feed, below header
**Trigger:** "+ Add something for the band" button (dashed border, always visible)
**Menu:** Poll / Decision, Idea, Note
**Forms:**
- **Poll**: Question + comma-separated options → saves to `bandPath('polls')` with `tag: 'needs_input'`
- **Idea**: Title + optional link → saves to `bandPath('ideas/posts')` with `tag: 'needs_input'`
- **Note**: Text only → saves to `bandPath('ideas/posts')` with `tag: 'fyi'`

After posting: toast "Posted", form resets, feed reloads to show new item.

## Time + Urgency

**Urgency tags on feed cards:** "⏰ Needed for upcoming gig" (red) or "⏰ Needed for next rehearsal" (yellow) when events are imminent.

**Sort bias:** Within same priority bucket, items near events sort higher.

**Rehearsal blocker:** Lock In dashboard shows "⚠️ N items still blocking rehearsal" when rehearsal is within 3 days.
