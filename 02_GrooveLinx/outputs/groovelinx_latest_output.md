# GrooveLinx — Spotify Match Review UX Upgrade

_Generated by Claude · 2026-03-24_

## Summary

Review experience redesigned to feel like improvement, not interruption. Choice before opening Spotify, live progress, re-sync button, skip option, better copy.

## Files Changed

| File | What |
|------|------|
| `js/core/listening-bundles.js` | Sync choice overlay, progress bar, re-sync, skip, copy improvements |

## UX Changes

### 1. Timing — Choice Before Opening
**Before:** Spotify opened immediately, review appeared 1.5s later
**After:** If failed songs exist, shows choice overlay first:
- "12 songs ready / 3 need review"
- [Fix matches] → opens review overlay
- [Open anyway] → opens Spotify playlist

### 2. Progress Indicator
Inside review overlay: live progress bar
- `[████░░] 1/3 fixed`
- Updates after each fix or skip
- Bar turns green when all handled
- Shows skip count: "2/3 fixed (1 skipped)"

### 3. Re-Sync Button
After all songs handled (fixed or skipped):
- "✅ All matches fixed"
- [🔄 Re-sync playlist] button
- Triggers `syncToSpotify()` again with locked IDs
- Overlay closes, sync runs with improved matches

### 4. Copy
**Header:** "🔍 Review Spotify Matches" → "🎵 Fix your playlist"
**Subtext:** "We'll remember your choices for next time."
**Toast on fix:** "✅ Locked for future syncs"

### 5. Skip Option
Each song card has a "Skip" button:
- Marks as skipped (dimmed, ⏭ icon)
- Does NOT lock a track — stays in failed state for next sync
- Counts toward handled total (progress advances)
- Useful when song doesn't exist on Spotify

### 6. All-Matched Path
If sync has 0 failures: opens Spotify immediately with toast "N songs synced — opening Spotify" (unchanged, no choice overlay).

## Flow Summary

```
syncToSpotify()
  ├─ 0 failed → open Spotify + toast
  └─ N failed → show choice overlay
                  ├─ "Fix matches" → review overlay
                  │     ├─ Search → select → lock ✅
                  │     ├─ Skip ⏭
                  │     └─ All handled → "Re-sync playlist" button
                  └─ "Open anyway" → open Spotify with partial playlist
```

## Edge Cases

- **All skipped:** Progress completes, re-sync available but will still fail those songs
- **Fix then re-sync:** Previously locked tracks matched instantly, only remaining unknowns searched
- **Close overlay:** Can re-open via `ListeningBundles.showReviewAfterSync(result)`
- **No Spotify token during review search:** Returns empty results, doesn't crash
