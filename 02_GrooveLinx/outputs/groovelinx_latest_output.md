# GrooveLinx — Review & Fix Spotify Matches

_Generated by Claude · 2026-03-24_

## Summary

After Spotify sync, if songs failed to match, a review overlay auto-appears. Users search Spotify, select the correct track, and lock it for all future syncs.

## Files Changed

| File | What |
|------|------|
| `js/core/listening-bundles.js` | Review overlay, Spotify search, track locking, auto-show after sync |

## Review UI

### Entry Point
After sync with failed songs (1.5s delay to let Spotify open):
- Modal overlay appears: "🔍 Review Spotify Matches"
- Lists each failed song with a "Search" button

### Per-Song Flow
1. User taps **Search** → queries Spotify API for 5 candidates
2. Each result shows: track name, artist, album
3. User taps a result → **"✅ Locked"** confirmation
4. Track ID saved to song's `spotify_versions` with `spotifyMatchLocked: true`
5. Toast: "✅ Track locked for future syncs"

### Custom Search
If initial search returns nothing:
- Shows editable text input pre-filled with song title
- User can modify search query and retry

### Completion
When all failed songs are fixed:
- Toast: "All matches fixed — sync again for updated playlist"
- Overlay stays open (user can close via ✕ or tap outside)

### Future Sync Behavior
`resolveSpotifyTrackIds()` checks locked IDs first (priority 1):
1. **Locked** (`spotifyMatchLocked: true`) → use `spotifyTrackId`, skip search
2. **URL extraction** → extract track ID from saved version URL
3. **API search** → search Spotify as fallback
4. **Fail** → add to `failedSongs`

## Integration with Sync

`syncToSpotify()` now:
- Stores result in `_lastSyncResult` for later access
- Auto-shows review overlay 1.5s after sync if `failed > 0`
- Toast changes: "12 songs synced, 3 need review"

## Edge Cases

- **No Spotify token**: Search button returns empty; review still shows song list
- **Track already locked**: `lockSpotifyTrack` skips duplicate, returns true
- **Save fails**: Shows "Failed to save — try again" on the item
- **Zero failed**: Review overlay never appears; toast shows only "N songs synced"
- **User closes overlay**: Can re-open via `ListeningBundles.showReviewAfterSync(result)`
- **Custom search**: Users can edit the query if the song title doesn't match Spotify's catalog
