## Fallback Search — Artist Mismatch Fix

### Root Cause
All fallback search queries (YouTube, Spotify, Archive) hardcoded `+ " Grateful Dead"` after the song title. **502 of 585 songs** were searching with the wrong artist.

### Impact by Band

| Band Code | Songs | Before (search query) | After |
|-----------|-------|-----------------------|-------|
| GD | 83 | ✅ "Grateful Dead" | ✅ Same |
| JGB | 19 | ❌ "Grateful Dead" | ✅ "Jerry Garcia Band" |
| ABB | 70 | ❌ "Grateful Dead" | ✅ "Allman Brothers Band" |
| Phish | 139 | ❌ "Grateful Dead" | ✅ "Phish" |
| WSP | 118 | ❌ "Grateful Dead" | ✅ "Widespread Panic" |
| DMB | 89 | ❌ "Grateful Dead" | ✅ "Dave Matthews Band" |
| Goose | 67 | ❌ "Grateful Dead" | ✅ "Goose" |

### Fix
New `_buildSearchQuery(songTitle)` in listening-bundles.js:
1. Looks up song in `allSongs` for `artist` and `band` fields
2. Maps band codes via `bandMap`: GD, JGB, ABB, Phish, WSP, DMB, Goose
3. Falls back to title-only if no artist/band found
4. Console logs every query for debugging

Fixed in 5 locations across 2 files:
- `listening-bundles.js`: play confirmation, quick launch fallback, Spotify search
- `playback-session.js`: fallback overlay
- `setlist-player.js`: queue builder band name mapping
