# GrooveLinx — Responsibility Targeting

_Generated by Claude · 2026-03-24_

## Summary

Items can now target specific band members or everyone. The action engine respects targeting — only targeted members see items in "I Owe". Non-targeted members see it as "Waiting on band".

## Files Changed

| File | What |
|------|------|
| `js/core/feed-action-state.js` | `_isTargetedAtMe()` check in `getActionState()`, targeting documentation |
| `js/features/band-feed.js` | Target selector in creation form, target data on items, "For: names" display |

## Data Structure

Stored in Firebase alongside existing item fields:

```json
{
  "question": "Should we open with Shakedown?",
  "options": ["Yes", "No", "Maybe"],
  "votes": {},
  "author": "Drew",
  "ts": "2026-03-24T...",
  "tag": "needs_input",
  "targetType": "specific",
  "targetMembers": ["chris", "brian"]
}
```

| Field | Values | Default |
|-------|--------|---------|
| `targetType` | `'all'`, `'specific'`, `'fyi'` | `'all'` |
| `targetMembers` | Array of member keys | `[]` |

**Backward compatible:** Items without `targetType` default to `'all'` — all existing items behave exactly as before.

## UI Updates

### Creation Form — Target Selector
**Location:** Below input fields, above Post/Cancel buttons (polls and ideas only, not notes)
**Controls:**
- "Everyone" button (default, highlighted)
- "Specific people" button → expands member checklist
- Member checklist shows all band members except self, with checkboxes

### Feed Card — "For: names"
**Location:** Below card body, above "Waiting on" line
**Trigger:** `targetType === 'specific'` and `targetMembers.length > 0`
**Display:** "For: Chris, Brian" (indigo text) — max 3 names, then "+N more"

## Action Engine Integration

`_isTargetedAtMe(item)` in FeedActionState:
- `targetType === 'all'` or missing → returns `true` (everyone)
- `targetType === 'fyi'` → returns `false` (no action for anyone)
- `targetType === 'specific'` → checks `targetMembers` against `getMyMemberKey()` and `getMyDisplayName()`

Effect on `getActionState()`:
- **Polls targeted at me:** `needsMyInput = !iVoted` (same as before)
- **Polls NOT targeted at me:** `needsMyInput = false`, shows as "Waiting on band"
- **Ideas targeted at me:** `needsMyInput = true`
- **Ideas NOT targeted at me:** `needsMyInput = false`

## Edge Cases

- **No targetType field (legacy items):** Defaults to `'all'` — backward compatible
- **targetMembers empty with targetType='specific':** Treated as `'all'` (defensive)
- **Member key vs display name:** `_isTargetedAtMe` checks both formats
- **Self excluded from member list:** Creation UI doesn't show current user in checklist
- **Notes:** Always FYI, no target selector shown
- **Quick add:** Posts as FYI note, no targeting

## Smoke Tests

1. Create poll → "Everyone" default → all members see it in "I Owe"
2. Create poll → select "Specific people" → check Chris, Brian → only Chris and Brian see it in "I Owe"
3. Drew creates poll targeting Chris → Drew sees it as "Waiting on band", Chris sees it in "I Owe"
4. Feed card shows "For: Chris, Brian" on targeted items
5. Legacy items (no targetType) still work as "everyone"
6. Notes have no target selector
7. Badge count only includes items targeted at current user
