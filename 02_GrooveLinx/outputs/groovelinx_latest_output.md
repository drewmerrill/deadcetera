## Trigger Conditions

"🎶 Your first GrooveLinx playlist is ready" fires when:
- `localStorage('gl_first_playlist')` is NOT set
- AND a sync completes with 0 failed songs

**Path 1 — Clean sync:** All songs match on first try → fires immediately
**Path 2 — Fix then re-sync:** First sync has failures → user reviews → fixes matches → re-syncs → 0 failures → fires on re-sync

## Logic

```
_showPlaylistReady(bundleType, matchedCount, playlistId)
  ├─ Check localStorage('gl_first_playlist')
  ├─ If not set: "🎶 Your first GrooveLinx playlist is ready" + set flag
  └─ If set: "{Type} playlist ready — N songs — stay locked in"
```

Called from:
1. `syncToSpotify()` when `resolved.failed === 0`
2. (Indirectly) `_resyncAfterReview()` → calls `syncToSpotify()` → same path

## Re-sync Toast Conflict Fix

**Problem:** `_resyncAfterReview` showed its own "All songs fixed" toast 2s after sync, overwriting the first-time moment.

**Fix:** Captures `gl_first_playlist` state before calling `syncToSpotify()`, checks if it changed after. If the first-time moment just fired, skips the improvement toast entirely.

```javascript
var hadFirstPlaylist = !!localStorage.getItem('gl_first_playlist');
var result = await syncToSpotify(...);
var firstJustFired = !hadFirstPlaylist && !!localStorage.getItem('gl_first_playlist');
if (!firstJustFired) { /* show improvement toast */ }
```

## Edge Cases

- **First sync clean:** First-time fires immediately, no re-sync needed
- **First sync with failures, then re-sync clean:** First-time fires on re-sync, improvement toast suppressed
- **First sync with failures, re-sync still has failures:** First-time doesn't fire (still has failures), improvement toast fires normally
- **Second+ sync:** Standard type-specific toast, first-time never fires again
