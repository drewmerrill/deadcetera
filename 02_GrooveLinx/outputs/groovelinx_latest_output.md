# GrooveLinx — Spotify State + Playback Dead-End Fixes

_Generated by Claude · 2026-03-24_

## Summary

Fixed two real UX bugs: Spotify button not reflecting connected state, and Play Now / Quick Launch ending in dead-end loading states.

## Root Causes

### Bug 1 — Spotify Button State
**Cause:** Dashboard listening card always rendered "Sync to Spotify" regardless of connection state. After OAuth callback, `handleSpotifyCallback` saved the token but the dashboard was already rendered with the old state and never re-rendered.

**Fix:**
- Button now checks `isSpotifyConnected()` at render time: shows "Connect Spotify" or "Sync to Spotify"
- After OAuth callback, `renderHomeDashboard()` is triggered (500ms delay) to re-render with connected state

### Bug 2 — Play Now Dead-End
**Cause:** `quickLaunch()` had two dead-end paths:
1. When `result.matched === 0`: showed toast "Finding best version…" then **returned without any UI** — user hung forever
2. When `result.matched > 0`: called `launchBundle()` → `window.open()` — **popup-blocked on iPad/Safari** because it was in an async context, not a direct user gesture

**Fix:**
- Zero matches: shows explicit fallback overlay with YouTube / Spotify / Archive buttons
- Successful matches: opens URL via short `setTimeout(100)` to keep closer to user gesture context
- Setlist player: `_showSearching()` now has 10-second timeout — if Invidious search hangs, automatically transitions to fallback UI
