## Band Completion Calculation

`FeedActionState.computeBandAlignment(items, metaMap)` returns:
```
{ pct, actionable, resolved, label }
```

- `actionable` = items with tag `needs_input` or `mission_critical` (not archived)
- `resolved` = items where `isResolved || completeForBand`
- `pct` = `round(resolved / actionable * 100)`, 100% if no actionable items
- `label` = "Locked in" (100%) / "Almost there" (75%+) / "Making progress" (50%+) / "Needs work" (<50%)

## Where UI Is Added

### 1. Feed Card — "Waiting on: names"
**Location:** Below poll body text, above inline vote buttons
**Trigger:** Poll where `state.waitingOnOthers === true`
**Display:** "Waiting on: Chris, Jay" or "Waiting on: Chris, Jay +1 more" (max 2 names shown)
**Source:** `FeedActionState.getWaitingMembers(item)` — checks `bandMembers` against `pollVotes` keys

### 2. Lock In Dashboard — Band Alignment Card
**Location:** Below "You Owe" card, above session plan
**Display:**
- Large percentage with colored progress bar
- Label: "Locked in" / "Almost there" / "Making progress" / "Needs work"
- "3/5 decisions resolved" when incomplete
- Green border + "Locked in" at 100%

**Data source:** Uses feed cache if available, otherwise lightweight poll-only Firebase query (same async pattern as Action Owed card)

## Color Coding

| Pct | Color | Label |
|-----|-------|-------|
| 100% | Green (#86efac) | Locked in |
| 75%+ | Indigo (#818cf8) | Almost there |
| 50%+ | Yellow (#fbbf24) | Making progress |
| <50% | Red (#f87171) | Needs work |

## Edge Cases

- **No actionable items**: Shows 100% "Locked in" (nothing pending = aligned)
- **Feed not yet loaded**: Falls back to lightweight poll query (same as badge refresh)
- **bandMembers not defined**: `getWaitingMembers()` returns empty array, no "Waiting on" line shown
- **All members voted**: No "Waiting on" line (state.waitingOnOthers is false)
- **Offline**: Card shows "Offline" fallback text
