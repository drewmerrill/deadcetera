# GrooveLinx — Playback Flow Reinforcement

_Generated by Claude · 2026-03-24_

## Summary

Playing indicator, return-from-Spotify detection, flow reinforcement toast, chart + playback connection with active song highlighting.

## Files Changed

| File | What |
|------|------|
| `js/core/listening-bundles.js` | Playing state, window focus return, flow toasts |
| `js/features/charts.js` | highlightActiveSong(), data-song attributes on setlist chart items |

## 1. Playing Indicator

When user taps play (embed CTA or open button):
- **"● Playing (in Spotify): Althea"** shown in green
- Appears in play overlay (if still open) and next-song bar
- Cleared when: Next Song tapped, or overlay closed

Not based on actual Spotify playback state — triggered by user action only.

## 2. Return from Spotify

Window `focus` event listener activated after play:
- When user returns to GrooveLinx: **"Back in GrooveLinx — tap Next to continue"** toast
- Fires once per play action (listener removes itself after firing)
- Only active while `_qlPlaying === true`

## 3. Flow Reinforcement

After first play, 4-second delay:
- **"Next up ready 👇"** toast
- Points user to the next-song bar at the bottom
- Only fires when there are multiple songs in the bundle

## 4. Chart + Playback Connection

### `ChartSystem.highlightActiveSong(songTitle)`
- Clears previous highlights
- Finds matching `[data-song]` element in setlist chart mode
- Adds: green left border (3px), background tint, **"● Now playing"** label
- Scrolls into view (smooth, centered)
- Call with `null` to clear all highlights

### Setlist Chart Mode
- Each song div now has `id="chart_song_{idx}"` and `data-song="{title}"`
- Ready for external highlighting from playback or setlist player
